## Codebase Patterns

### Design system (carried forward from PRD-01 — non-negotiable)
- DESIGN.md (repo root) is the visual source of truth; read it before ANY UI work, every Ralph loop. Consume tokens, never hardcode color/type/spacing/radius/shadow. Web = design/tokens.css (Tailwind v4 @theme, use var(--token)); RN = design/theme.ts. Add missing values to DESIGN.md + both token files FIRST, then consume. Enforced in CLAUDE.md + AGENTS.md + the checklist.md reviewer gate — reject any change that hardcodes a visual value or diverges from DESIGN.md.
- Brand fonts are self-hosted under design/fonts/ (inter/, space-grotesk/, licenses). Web loads them via design/fonts.css (@font-face families "Inter" + "Space Grotesk"), which must be imported so the --font-* tokens resolve. NEVER hardcode a font family — the @font-face name IS the --font-display / --font-sans token value.
- Design invariants (verified against Partiful source, HOLDS TRUE): black/white action system (no colored buttons); gradients live on surfaces/overlays ONLY, never on buttons/interactive elements (verify CTA computed backgroundImage:none); max two type sizes per card; Space Grotesk display face only for hero headlines / largest titles (≥26px) — oversized numerals use Inter, not the display face; Warm Sand (#d9c58b) only for active states; status colors only on generation-state Badge dots, never decorative; selection = 2px midnight-ink outline ring (no reflow) or border-color swap; labels over photos = Badge `onPhoto` + --gradient-scrim overlay.
- Token-literal audit before marking any UI story done: `grep -rnE '#[0-9a-fA-F]{3,8}|font-family.*(inter|space|sans-serif)|font-size:.*px' <component/css files>`. Only allowed literals: 1px/2px border & focus-outline widths, opacity multipliers, transforms, transition timing, @keyframes. Everything visual else must be a token.

### Analytics / conversion tracking (established US-012)
- **Analytics seam lives in `web/app/lib/analytics.ts`** (client-safe, no server imports) — the single place the site fires product events, same swap pattern as `saveWaitlistEmail` / `PRIMARY_CTA`. Exposes `track(event, props)` + a small `EVENTS` map (`ctaClick`, `waitlistSignup`). To change providers, keep the event names + `track` signature and re-point the body; callers (`CtaLink`, `Waitlist`) never move. Any new tracked event goes in `EVENTS` first, then callers reference it — never a raw string.
- **Provider = Plausible** (cookieless, no PII, no cross-site tracking → GDPR/CCPA-friendly, needs no consent banner). Loaded by `web/app/components/Analytics.tsx` (rendered in the root layout) via `next/script` ONLY when `analyticsEnabled` = `process.env.NODE_ENV === "production" && NEXT_PUBLIC_PLAUSIBLE_DOMAIN` set. Includes the recommended inline queue stub (`window.plausible.q`) so events fired before the deferred script loads are buffered + flushed. Env: `NEXT_PUBLIC_PLAUSIBLE_DOMAIN` (site domain) + `NEXT_PUBLIC_PLAUSIBLE_SRC` (optional script-url override for self-host/proxy) — documented in AGENTS.md.
- **No dev noise + consent**: with no provider loaded, `track()` logs `[analytics] <event> <props>` via `console.debug` (the dev verification path) and makes ZERO network requests — the Plausible script is never injected in dev. `track()` also honors Do Not Track / Global Privacy Control (`navigator.doNotTrack`/`msDoNotTrack`/`globalPrivacyControl`) and no-ops when the visitor has opted out (verified: DNT on → 0 events even with the script loaded).
- **Wiring pattern**: primary CTA clicks go through `web/app/components/CtaLink.tsx` — a client wrapper over `ButtonLink` that reads the `PRIMARY_CTA` swap seam and fires `ctaClick {location, mode}` on click. Use `<CtaLink location="…" />` for EVERY primary CTA (header, hero, mobile-menu) instead of a bare `<ButtonLink href={PRIMARY_CTA.href}>` so the click is tracked and the swap seam stays centralized. Rendering this client leaf inside a server component (Hero) is fine — only the interactive leaf ships JS. Conversion events (e.g. `waitlistSignup`) fire from the SUCCESS branch of the action (after the server 200), never on attempt, so spam/invalid submissions don't inflate conversions.
