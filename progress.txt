## Codebase Patterns

### Design system (carried forward from PRD-01 — non-negotiable)
- DESIGN.md (repo root) is the visual source of truth; read it before ANY UI work, every Ralph loop. Consume tokens, never hardcode color/type/spacing/radius/shadow. Web = design/tokens.css (Tailwind v4 @theme, use var(--token)); RN = design/theme.ts. Add missing values to DESIGN.md + both token files FIRST, then consume. Enforced in CLAUDE.md + AGENTS.md + the checklist.md reviewer gate — reject any change that hardcodes a visual value or diverges from DESIGN.md.
- Brand fonts are self-hosted under design/fonts/ (inter/, space-grotesk/, licenses). Web loads them via design/fonts.css (@font-face families "Inter" + "Space Grotesk"), which must be imported so the --font-* tokens resolve. NEVER hardcode a font family — the @font-face name IS the --font-display / --font-sans token value.
- Design invariants (verified against Partiful source, HOLDS TRUE): black/white action system (no colored buttons); gradients live on surfaces/overlays ONLY, never on buttons/interactive elements (verify CTA computed backgroundImage:none); max two type sizes per card; Space Grotesk display face only for hero headlines / largest titles (≥26px) — oversized numerals use Inter, not the display face; Warm Sand (#d9c58b) only for active states; status colors only on generation-state Badge dots, never decorative; selection = 2px midnight-ink outline ring (no reflow) or border-color swap; labels over photos = Badge `onPhoto` + --gradient-scrim overlay.
- Token-literal audit before marking any UI story done: `grep -rnE '#[0-9a-fA-F]{3,8}|font-family.*(inter|space|sans-serif)|font-size:.*px' <component/css files>`. Only allowed literals: 1px/2px border & focus-outline widths, opacity multipliers, transforms, transition timing, @keyframes. Everything visual else must be a token.

### Landing site (web/) — established US-001
- The marketing site is a Next.js 16 App Router + React 19 + TS + Tailwind v4 project at `web/`. Sharing mechanism for the design system is **direct relative `@import`, NOT copy/package**: `web/app/globals.css` does `@import "tailwindcss"; @import "../../design/fonts.css"; @import "../../design/tokens.css";`. This keeps `design/` the single source of truth — never copy token values into `web/`.
- CRITICAL wiring: `web/next.config.ts` sets `turbopack.root` to the repo root (one level up from `web/`). Without it, Turbopack won't resolve the parent-dir `../../design/*` CSS imports or bundle the `design/fonts/*` url()s. Keep it.
- Consume tokens in web components via `var(--token)` inline styles OR Tailwind utilities generated from the `@theme` block. Inline `var()` is clean and unambiguous and passes the token-literal audit — prefer it for one-off page styling; port `components.css` classes when bringing over PRD-01 components (pseudo-states need real CSS).
- Web quality gate: `cd web && npm run typecheck && npm run lint && npm run build`. Lint uses ESLint flat config importing `eslint-config-next/core-web-vitals` + `/typescript` directly — `next lint` is REMOVED in Next 16, and `FlatCompat` + eslint-config-next crashes on ESLint 9.39 ("circular structure to JSON"), so use the native flat exports.
- Browser verify: `cd web && npm run dev` (port 3100), then drive with the temp Playwright/system-Chrome pattern. Verify webfonts programmatically after `document.fonts.ready`: `document.fonts.check('14px Inter')` + `document.fonts.check('36px "Space Grotesk"')`.
